    1、事务的特性

    事务

    什么是事务？

    数据库事务是指作为单个逻辑工作单元执行的一系列SQL操作。

    这些操作要么全部执行，要么全部不执行。

    就像git一样，commit之后要么全部push，要么roll back。

    为什么需要事务？

    就像银行转账一样，甲转账给乙10元。数据库操作需要两步。①甲账户减去10元②乙账户增加十元。如果没有事务并且两步中间发生异常，就会导致数据不同步。

    事务管理是每个数据库（Oracle、MySql、DB等）都必须实现的。

    事务有哪四种特性？

    事务的特性有ACID四种，看下文。

    事务的运行模式有哪三种？

    1. 自动提交事务：默认事务管理模式。如果一个语句成功地完成，则提交该语句；如果遇到错误，则回滚。
    2. 显性事务：以Begin Transaction显式开始，以Commit或者RollBack显式结束。
    3. 隐性事务：当连接以此模式进行操作室，Sql将在提交或回滚当前事务后自动启动新事务。无需描述事务的开始，只需提交或回滚每个事务。它会自动的生成连续的事务链。

    事务的隔离性？

    当多个线程都开启事务操作数据库时，数据库要能进行隔离操作，以此来确保各个线程获取数据的准确性。

    若没有隔离，则可能发生以下问题

    1、脏读

    一个事务正在多次修改某个数据，而这个事务的多个修改还未全部提交。

    此时一个并发的事物来访问该数据，就会造成两个事务得到的数据不一致。

    脏读是某一事务读取了另一事务未提交的脏数据。

    2、不可重复读

    事务A要多次读取了某一数据。

    事务A某一次读取后的同时，事务B立马修改了这个数据并提交。

    事务A再次读取该数据就得到了不同结果。

    不可重复读是读取了前一事务提交的数据。

    在某些情况下不可重复读并不是问题，我们多次查询某个数据当然以最后查询得到的结果为主。

    3、虚读/幻读

    事务A在对某个数据项进行”1”修改为”2”的操作时，事务B又在此表中插入了一行数据，并且包括数据项”1”。

    此时A再查看数据会发现有一行的”1”没有修改为”2”，但其实是B添加的，就像幻觉一样，这就是幻读。

    幻读和不可重复读都是读取了另一条已经提交的事物。

    但是幻读是设计一整行数据。

    MySqL的四种隔离级别

    ① Serializable

    Serializable (串行化)：可避免脏读、不可重复读、幻读的发生。

    ② Repeatable read

    Repeatable read (可重复读)：可避免脏读、不可重复读的发生。

    ③ Read committed

    Read committed (读已提交)：可避免脏读的发生。

    ④ Read uncommitted

    Read uncommitted (读未提交)：最低级别，任何情况都无法保证。

    以上四种隔离级别最高的是 Serializable 级别，最低的是 Read uncommitted 级别，当然级别越高，执行效率也会越低。

    像 Serializable 这种级别，就是以锁表的方式让其他线程等待。

    我们选用那种隔离级别应该根据实际情况来进行，在 MySqL 中默认的隔离级别是 Repeatable read (可重复读)。

    2、ACID是指

    ACID是事务的四种特性。

    • Atomicity原子性：强调事务不可分割。
    • Consistency一致性：事务执行前后数据的完整性保持一致。
    • Isolation隔离性：一个事务执行过程中，不会收到其他事务的干扰。多个事务不会同时进行，同时提供了多种隔离级别。
    • Durability持久性：事务一旦提交完成，对数据库的改变是永久性的。即使数据库异常也不会丢失事务。

    3、C++的重载

    C++允许在同一个作用域中的某个函数和运算符指定多个定义，分别称为函数重载和运算符重载。

    重载声明是指一个和之前已经在该作用域内声明过的函数或方法具有相同名称的生命，但是他们的参数列表和定义（实现）不同。

    当你调用一个重载函数/重载运算符时，编译器通过把您所使用的的单数类型和定义中的参数类型进行比较，决定最合适的定义。

    而这个选择最合适的过程，称为重载决策。

    1、函数重载

    在同一个作用域内，可以声明几个功能类似的同名函数，但是这些同名函数的形式参数（指参数的个数、类型或者顺序）必须不同。

    您不能仅通过返回类型的不同来重载函数。

    2、运算符重载

    您可以重定义或重载大部分 C++ 内置的运算符。这样，您就能使用自定义类型的运算符。

    重载的运算符是带有特殊名称的函数，函数名是由关键字 operator 和其后要重载的运算符符号构成的。与其他函数一样，重载运算符有一个返回类型和一个参数列表。

    声明加法运算符用于把两个 Box 对象相加，返回最终的 Box 对象。大多数的重载运算符可被定义为普通的非成员函数或者被定义为类成员函数。如果我们定义上面的函数为类的非成员函数，那么我们需要为每次操作传递两个参数，如下所示：

    1、可重载运算符

    双目算术运算符 + (加)，-(减)，*(乘)，/(除)，% (取模)
    关系运算符 ==(等于)，!= (不等于)，< (小于)，> (大于)，<=(小于等于)，>=(大于等于)

    4、视图的作用和优点

    视图概述

    当用户对数据库中一张或多张表的某些字段的组合感兴趣，而又不想每次键入这些查询时，用户就可以定义一个视图来解决这个问题。

    视图中的列可以来自表中的不同列，这些列都是用户所感兴趣的数据列。

    视图和表不一样，视图在物理上不存在，是一个虚表。在数据库中只存放视图的定义，而不存放视图对应的数据。

    数据是放在原本的表中。表中的数据变化了，视图中查出的数据也会变化。每一次查看视图或引用视图就会运行一次视图上的查询。

    SELECT可以查询视图数据，对于符合约束条件的视图，也可以用 INSERT、UPDATE、DELERE、MERGE INTO 来修改视图对应的表中的数据。

    视图在提供操作方便的同时，还可以保障数据库数据的安全。

    视图的优点

    1、视图可以简化复杂的查询

    多表联合查询的SQL通常很复杂，通过视图，就可以使用简单的SQL来规避复杂的连接语句。

    2、视图可限制用户的访问

    可以对不同用户提供不同视图，公开不同的数据。

    3、视图提供了额外的安全层

    数据库可以创建只读视图，这样用户可以在视图中检索数据，却不能修改数据。

    4、视图可以定义计算列

    视图可以额外定义一个列，来动态的计算不同列中的数据。

    例如商品数量和商品价格。我们可以定义一个总销售额列，那么这个列会根据数量和价格自动动态计算。

    5、视图支持应用兼容性

    当我们要重新设计数据库的时候，可能会删除一些表，创建一些表，修改表的列名。那么在这种情况下，可以使用和旧表结构相同的视图，应用程序可以访问视图来完成之前的功能，就不需要修改应用程序。

    5、数据库管理系统的主要功能

    数据库管理系统 DBMS 是位于 OS 和用户之间的一个数据管理软件

    1. 数据定义功能。DBMS 提供相应数据语言来定义（DDL）数据库结构，它们是刻画数据库框架，并被保存在数据字典中。
    2. 数据操作功能。DBMS 提供数据操纵语言（DML），实现对数据库数据的基本存取操作：检索，插入，修改和删除。
    3. 数据库的事物管理和运行管理功能。这是 DBMS 的核心部分，它包括并发控制，安全性检查，完整性约束，故障恢复等功能。DBMS提供数据控制功能，即是数据的安全性、完整性和并发控制等对数据库运行进行有效地控制和管理，以确保数据正确有效。
    4. 数据库的建立和维护功能。包括数据库初始数据的装入，数据库的转储、恢复、重组织，系统性能监视、分析等功能。
    5. 数据库的传输。DBMS提供处理数据的传输，实现用户程序与 DBMS 之间的通信，通常与操作系统协调完成。

    6、什么是this指针，什么时候用

    this 指针概述

    this 是 C++ 中的一关键字，也是一个 const 指针，它指向当前对象，类型是当前 class* 类型。通过他可以访问当前对象的所有成员。

    所谓当前对象是指正在使用的实例化对象。

    this 只能用在类的内部，通过 this 指针可以访问类的所有成员，包括 private、protected、public 属性。

    当成员函数中的形参和成员变量重名，就要用 this 进行区分。this -> name = name。（this是指针，要用 -> 来访问）

    this 虽然在类的内部，但只有类被创建之后才会给 this 赋值，而这个过程是编译器自动完成的，用户无法修改。不同的对象，this 值也不一样。

    • this 是 const 指针，他的值不能修改，一切企图修改的操作都是不允许的。
    • 只有在对象被创建后 this 才有意义，因此不能在 static 成员函数中使用。
    • this 只能在成员函数内部使用，在其他地方使用是没意义的，也是非法的。（介于此点，成员函数定义中可以获取实例化之后的信息。）

    this 究竟是什么？

    this 实际上是成员函数的一个形参，在调用成员函数时，会将对象的地址作为实参传给 this。

    但这个 this 形参是隐含的，不出现在代码中，由编译器完成。

    this 作为隐式形参，本质上是成员函数的局部变量。因此只能用在成员函数内部，并且只有在通过对象调用成员函数时才赋值 this。

    7、关系数据库的特点？

    关系数据库（Relational Database）是创建在关系模型基础上的数据库。

    关系模型可以简单理解为二维表格模型，关系型数据库就是由二维表和他们之间的关系组层的一个数据组织。

    优点

    严格建立在数学基础上

    关系模型概念单一

    关系模型的存取路径对用户透明，从而有更高的数据独立性，更好的安全性，也简化了程序员的工作和开发

    缺点

    由于存储路径透明，效率不如非关系模型。

    8、什么是层次模型、网状模型？

    层次模型

    层次模型是数据库系统中最早出现的数据模型，层次数据库系统采用层次模型作为数据的组织方式。它采用树形结构来表示各类实体以及实体间的联系。

    就像树一样。

    层次模型的特点：任何一个给定的记录值只能按其层次路径查看，没有一个子女记录值能够脱离双亲记录值而独立存在。

    1）优点：数据结构比较简单清晰，数据库的查询效率高，提供了良好的完整性支持。
    2）缺点：现实世界中很多联系是非层次性的，它不适用于结点之间具有多对多联系；查询子女结点必须通过双亲结点；由于结构严密，层次命令趋于程序化。

    网状模型

    网状数据模型的典型代表是DBTG（CODASYL）系统。

    满足一下两个条件的基本层次联系的集合为网状模型
    1、允许一个以上的结点无双亲；
    2、一个结点可以有多于一个的双亲。
    备注：层次模型实际上是网状模型的一个特例。

    网状模型中子女结点与双亲结点的联系可以不唯一。

    3、网状模型的优缺点
    1）优点：能够更为直接地描述现实世界，如一个结点可以有多个双亲，结点直接可以有多种联系；具有良好的性能，存取效率较高。
    2）缺点：结构比较复杂，随应用环境的扩大，数据库的结构就变得越来越复杂，不利于最终用户掌握；网状模型的DDL、DML复杂，并且要嵌入某一种高级语言（C、COBOL）中，用户不容易掌握和使用；由于记录之间的联系是通过存取路径实现的，应用程序在访问数据时必须选择适当的存取路径，因此用户必须了解系统结构的细节，加重了编写应用程序的负担。
